# 강의 제목 : 대화 문맥 관리와 캐싱

## 용어 정리
### 캐싱 : 자주 조회되는 데이터를 미리 In-memory DB에 저장해서 확인하고, 없으면 DB를 조회하는 기법
### In-memory DB : 데이터를 보조기억장치가 아닌, 컴퓨터의 메인 메모리(RAM)에 저장하고 처리하는 데이터베이스 시스템
      -> 대표적인 것이 "Redis"

---

## 대화 문맥 관리의 필요성
- 사용자 답변에 따라서 점점 더 답변이 고도화 되고, 원하는 답변을 받을 수 있기 때문이다.

---

## In-memory DB와 캐싱의 개념
### Caching & Disk I/O
- 자주 사용하는 데이터는 빠르게 조회가 가능한 공간에 저장하여 디스크 부하를 감소시키고 성능을 최적화

## Redis
1. 엄청난 속도 (빠른 성능) -> 메모리에 저장을 하기 때문에
2. 다양한 데이터 구조 지원
  - 리스트 (List) : 큐와 스택 형태로 데이터를 저장
  - 셋 (Set) : 중복 없는 집합
  - 해쉬 (Hash) : 필드와 값을 묶어서 데이터를 저장
3. 단순한 사용법
